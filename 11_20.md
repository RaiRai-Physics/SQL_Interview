11. What is a CTE?

Answer:
CTE stands for Common Table Expression.
It is a temporary named result set defined using WITH.

Example:

WITH dept_avg AS (
    SELECT department_id, AVG(salary) AS avg_salary
    FROM employees
    GROUP BY department_id
)
SELECT *
FROM dept_avg;
12. What is the difference between a subquery and a CTE?

Answer:

A subquery is nested inside another query.
A CTE makes the query easier to read and reuse within the same statement.
Recursive queries are usually written using CTEs.
13. What is normalization?

Answer:
Normalization is the process of organizing data to reduce redundancy and improve data integrity.

Common normal forms:

1NF: atomic values, no repeating groups
2NF: no partial dependency
3NF: no transitive dependency
14. What is denormalization?

Answer:
Denormalization adds redundant data to improve read performance.
It is often used in analytics systems where query speed matters more than strict normalization.

15. What is an index?

Answer:
An index is a database object that improves the speed of data retrieval.
It works like a book index, helping the database find rows faster.

Downside: indexes can slow down INSERT, UPDATE, and DELETE.

16. What is the difference between clustered and non-clustered index?

Answer:

Clustered index determines the physical order of data in the table.
Non-clustered index stores pointers to the actual data.

A table usually has one clustered index, but can have many non-clustered indexes.

17. What is the difference between UNION and UNION ALL?

Answer:

UNION removes duplicate rows.
UNION ALL keeps duplicates and is faster.
18. What are aggregate functions?

Answer:
Aggregate functions perform calculations on multiple rows and return a single result.

Examples:

COUNT()
SUM()
AVG()
MIN()
MAX()
19. What is the difference between COUNT(*), COUNT(column), and COUNT(DISTINCT column)?

Answer:

COUNT(*) counts all rows
COUNT(column) counts non-null values in that column
COUNT(DISTINCT column) counts unique non-null values
20. What are window functions?

Answer:
Window functions perform calculations across a set of rows related to the current row without collapsing the result.

Examples:

ROW_NUMBER()
RANK()
DENSE_RANK()
SUM() OVER()
AVG() OVER()

Example:

SELECT name, department, salary,
       RANK() OVER (PARTITION BY department ORDER BY salary DESC) AS rank_in_dept
FROM employees;
